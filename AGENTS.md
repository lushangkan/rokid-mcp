Use Bun as the package manager.

When installing dependencies with Bun, use `bun install --no-cache`.

At the start of each task, review the project documentation to understand the project structure and dependencies.

Whenever you begin implementing a new feature, create a new branch named `feature/feature-name` using English only.

Every commit message must be clear, descriptive, and written in English using one of the following formats:

- `feat:` add a new feature
- `fix:` fix a bug
- `docs:` update documentation
- `style:` adjust code formatting without changing behavior
- `refactor:` restructure code without adding features or fixing bugs
- `test:` add or update tests
- `chore:` other changes such as build process or dependency management

It is correct for the `docs/` directory to remain untracked. Do not add it to version control.
